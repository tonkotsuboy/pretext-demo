01

パフォーマンス比較

「テキストが何行になるか、高さは何pxか」を知る2つの方法を比較します。

DOM計測（従来）

非表示のdivにテキストを入れて offsetHeight を読む。
ブラウザは毎回「レイアウトリフロー」（ページ全体の配置再計算）を強制実行するため、500回繰り返すと重い。

Pretext（Canvas計測）

prepare() でCanvas APIを使い文字幅を事前キャッシュ（初回のみ）。
layout() はキャッシュ済みの数値で足し算するだけ。DOMに一切触らないので桁違いに速い。

import { prepare, layout } from "@chenglou/pretext"

// Pretext: 2ステップで高さ取得
const prepared = prepare(text, '16px "Noto Sans JP"')  // 初回のみ
const { height, lineCount } = layout(prepared, 300, 28) // 算術計算のみ
02

Masonryレイアウト

DOMに触れずにカード高さを予測。リサイズしてみてください。

import { prepare, layout } from "@chenglou/pretext"

// カードの高さをDOMなしで予測
const prepared = prepare(text, '14px "Noto Sans JP"')
const { height } = layout(prepared, cardWidth, 24)
// → この height でMasonryの配置を計算
03

チャットバブル — fit-content比較

CSSの width: fit-content は最長行に合わせるだけ。Pretextは行数を維持したまま最小幅に縮める（shrinkwrap）。

import { prepare, layout } from "@chenglou/pretext"

// 行数を維持したまま最小幅を二分探索
const base = layout(prepared, maxWidth, 24)
let lo = 60, hi = maxWidth, bestW = maxWidth
while (lo <= hi) {
  const mid = (lo + hi) >> 1
  layout(prepared, mid, 24).lineCount <= base.lineCount
    ? (bestW = mid, hi = mid - 1) : (lo = mid + 1)
}

04

Canvas直接描画

layoutWithLinesで取得した行データをCanvasに直接描画。障害物の回り込みも。

import { prepareWithSegments, layoutNextLine } from "@chenglou/pretext"

// 1行ずつレイアウト。幅を変えれば障害物の回り込みが可能
const prepared = prepareWithSegments(text, font)
let cursor = { segmentIndex: 0, graphemeIndex: 0 }
while (true) {
  const width = isObstructed(y) ? narrowWidth : fullWidth
  const line = layoutNextLine(prepared, cursor, width)
  if (!line) break
  ctx.fillText(line.text, x, y)
  cursor = line.end; y += lineHeight
}
05

リアルタイムリフロー

スライダーで幅を変更。Pretextが瞬時に再計算。

import { prepare, layout } from "@chenglou/pretext"

// prepare()は初回のみ。layout()は何度呼んでも高速
const prepared = prepare(text, font)       // 一度だけ
const result = layout(prepared, width, 28) // ← 0.00xms で完了
